# Bright-coffee-shop-analysis
This project focuses on analyzing transactional sales data from a coffee shop to generate business insights for decision-making. The goal is to help a new CEO understand sales performance, identify trends, and improve revenue.

⸻

🎯 Objectives
	•	Identify top-performing products
	•	Analyze sales trends over time
	•	Determine peak sales hours
	•	Provide data-driven business recommendations

⸻

🛠️ Tools & Technologies
	•	Databricks – Data processing
	•	SQL – Data transformation and analysis
	•	Microsoft Excel – Data visualization (Pivot Tables & Charts)
	•	Miro – Data flow and architecture design
	•	PowerPoint – Presentation of insights

⸻
📂 Project Structure
	•	data/ → Raw and processed datasets
	•	sql/ → SQL scripts used for data cleaning and analysis
	•	visuals/ → Excel dashboards and charts
	•	presentation/ → Final PowerPoint presentation
	•	miro/ → Data architecture diagram

⸻

🔄 Data Processing Steps
	1.	Converted Excel dataset to CSV format
	2.	Cleaned inconsistent data (e.g., unit_price formatting)
	3.	Created calculated fields:
	•	total_amount = unit_price × transaction_qty
	•	time_bucket for time-based analysis
	4.	Stored processed data for analysis

⸻

📊 Key Analysis Performed
	•	Total revenue calculation
	•	Revenue by product
	•	Sales trends by time intervals
	•	Quantity sold per product

⸻

📈 Key Insights
	•	Top-performing products contribute the most to revenue
	•	Sales peak during specific hours of the day
	•	Some products sell frequently but generate lower revenue
	•	Opportunities exist to improve underperforming products

⸻

💡 Recommendations
	•	Increase stock for best-selling products
	•	Run promotions during low-sales periods
	•	Improve or replace underperforming items
	•	Optimize staffing during peak hours

⸻

🚀 Future Improvements
	•	Automate data pipeline and reporting
	•	Expand analysis across multiple store locations
	•	Implement customer loyalty programs
